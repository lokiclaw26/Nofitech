## TIGHT scope (~150 LOC total)

### 1. NEW: `src/components/ManualImageInput.tsx` (~80 LOC)

Mirror the existing `Dialog`/`Button`/`AnimatePresence`/`motion` pattern already used in `AddComponent.tsx`.

Props:
- `componentId: number`
- `currentImagePath?: string | null`
- `onSaved: (newImagePath: string) => void` — called after a successful upload
- `onCancel: () => void`

State:
- `tab: 'upload' | 'url'`
- `file?: File` (selected file)
- `filePreview?: string` (data URL for preview)
- `url: string` (typed URL)
- `urlPreview?: string` (fetched image as data URL)
- `busy: boolean`
- `error?: string`

UI:
- Dialog with two tabs (Upload / URL)
- Upload tab: file input (`accept="image/*"`), preview when file selected, "Save image" button (disabled if no file or busy)
- URL tab: text input for URL, "Preview" button (fetches via API, shows preview), "Save image" button
- After save: call `onSaved(newPath)`, parent closes the dialog
- Error toast / inline message for failures

Use `uploadImage(componentId, file)` and `setImageUrl(componentId, url)` from `src/lib/api.ts` (already exists).

### 2. MODIFY: `src/pages/AddComponent.tsx` (~50 LOC changes)

In the candidate list rendering (where search results are shown as cards):
- For each candidate, if `!candidate.image_url && !candidate.image_path` (no image anywhere):
  - Show "Add image" button (small button, e.g. `<Button variant="outline" size="sm">`)
- If image IS present, show the image (current behavior, unchanged)
- "Add image" button click → opens `<ManualImageInput>` modal in a controlled state
- After `onSaved` callback: update the candidate's image_path in local state, close modal

State to add:
- `imageModalOpen: boolean`
- `imageModalCandidate: Candidate | null`

Imports to add:
- `import { ManualImageInput } from "@/components/ManualImageInput"`
- `import { Button, Dialog, ... } from "<wherever they come from>"`
