| **acceptance_criteria** | AC1: New module `code/backend/app/quality.py` with 3 pure functions: `clean_candidates(raw_list)`, `score_candidate(c, query)`, `validate_image(c)`. AC2: `search()` calls these in order: clean → score → sort → filter (>= 0.50 confidence) → return. AC3: Hard blacklist terms in titles/descriptions: "tutorial", "demo", "library", "example", "project", "blog", "github.io", "guide", "how to", "course", "lesson", "workshop", "intro to", "getting started". Any candidate whose title or description contains any of these gets confidence multiplied by 0.0 (i.e. hard reject). AC4: Title must contain at least one meaningful query token (not just the bare letters of the query in random order). Tokens are: split query on whitespace, drop tokens < 3 chars, check at least one appears in title. AC5: Image must be present and pass a real-image check (extension jpg/jpeg/png/webp AND not on the placeholder blocklist). If image missing or fails: confidence -= 0.30. AC6: Source priority boost: Adafruit/PlatformIO/vendor += 0.10, manufacturer/wikidata+image += 0.10, GitHub/wikipedia -= 0.10, Wikimedia Commons without wikidata id stays neutral. AC7: Sorted by confidence DESC, capped at 12 results. AC8: Response shape now includes: `candidates` (cleaned+ranked), `rejected_count` (how many were filtered out), `quality_threshold` (the 0.50 cutoff used), `best_confidence` (max score in cleaned set, 0.0 if empty). AC9: Frontend Model Picker (AddComponent.tsx) renders: if `candidates.length === 0`, shows "No reliable match found. Try a different search or enter manually." with Enter Manually button. If `candidates.length > 0`, shows count + confidence badge per card + source attribution under each card. AC10: All 6 test cases return high-quality results. ESP32 DevKit V1, Wemos D1 Mini, BME280, INA219, TP4056 all return real product/datasheet candidates with images. ILI9488 XPT2046 returns a real datasheet/driver candidate (or "no reliable match" if no clean result). |
| **risks** | (1) Over-aggressive filtering might hide legitimate results. Mitigated by: keep "platformio.org/boards/..." and "adafruit.com/product/..." URLs always visible regardless of title heuristics, since those are trusted source pages. (2) Token-based title matching might miss legitimate abbreviations. Mitigated by: use normalized lowercase + collapse spaces. (3) The image placeholder check needs an explicit blocklist since SVG and external CDN URLs could be anything. Mitigated by: only accept jpg/jpeg/png/webp from approved domains (commons.wikimedia.org, cdn-shop.adafruit.com, platformio.org, raw.githubusercontent.com, github.com, *.jpg-only wikimedia thumbs). |
