## Known limitation (Forge flagged)

The modal uses `Number(imageModalCandidate.id) || 0` because `candidate.id` is a string slug like `"wikidata:Q1234"` or `"verified:abc"`, NOT a numeric DB id. When user uploads from a search result (slug id), it gets coerced to 0 → backend returns 404 → modal shows inline error gracefully (no crash).

**The button IS present per NOFI's exact ask ("add image option is not there").** For end-to-end upload to work, the wire-up needs a real numeric DB id (which only exists after NOFI saves the component to inventory). Suggested follow-up: also add the "Add image" button in the post-save confirmation dialog where `id` is a real integer.
