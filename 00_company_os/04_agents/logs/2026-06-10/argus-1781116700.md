## Bugs found
- **BUG-1 (MEDIUM): `renderProvider` in mission-control.html emits the `title="..."` string as a child of `<td>` instead of as an HTML attribute on the `<td>` itself.**
  - File: `01_projects/mission-control/code/mission-control.html`, function `renderProvider`, line 372
  - Current: `<td>${badge(cs, csCls)}${csTitle}</td>` where `csTitle = ' title="..."'`
  - Expected: `<td${csTitle}>${badge(cs, csCls)}</td>` (place attribute on the `<td>` before the badge, or apply it to the badge span)
  - Impact: The connection_detail string is rendered as visible text after the badge, instead of as a hover tooltip. The spec ("Connection cell uses title attribute for the detail string (not the visible text)") is not met. The user sees literal `title="..."` text in the Connection cell. Functionally the status badge is still readable, so the build is not broken — but the UX intent is violated and the detail is ugly in the cell.
