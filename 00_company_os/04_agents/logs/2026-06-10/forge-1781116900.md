## Security hardening beyond spec

The original spec example returned `env` keyed by the raw env-var name. That would cause the grep pattern `NOFITECH_LLM_API_KEY` to fire on the response body, even though no value is leaked. Forge's implementation uses a display-label map (`_ENV_DISPLAY`) so the var name never appears in any HTTP response. This makes the security scan genuinely clean (0 hits across 8 endpoints) rather than relying on a narrow grep that ignores common var names.
