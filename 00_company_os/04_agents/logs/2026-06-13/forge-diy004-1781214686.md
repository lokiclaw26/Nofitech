## How the New Image Pipeline Works

1. User types query like "ESP32 DevKit V1" in the single input
2. Frontend POSTs `{query: "ESP32 DevKit V1"}` to /api/components/search
3. Backend `parse_query` identifies the family (esp32) AND the specific model (DevKit V1), returns 1 candidate
4. Search endpoint attaches the hardcoded `image_url` (Wikimedia Commons thumbnail, pre-verified)
5. Frontend renders the model-picker dialog (skipped for 1 candidate) or confirmation popup with the real image
6. On save, the backend downloads the image to `data/images/<slug>.<ext>`, stores the local path

For families where the user doesn't specify a model (e.g. just "esp32"), the parser returns all family candidates, each with its own unique image, so the user can visually distinguish them in the picker.
