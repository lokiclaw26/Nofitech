## Files Modified

`code/frontend/src/pages/AddComponent.tsx` (CandidateImage function, lines 64-86):
- **Before:** `dangerouslySetInnerHTML={{ __html: svg }}` (renders the SVG at its natural 400×400, overflows the 200×200 container, gets clipped → image appears missing)
- **After:** A new `scaled` variable applies a regex to override ONLY the outer `<svg>` tag's `width` and `height` attributes to `"100%"` and `"100%"`, then renders the scaled SVG. Inner `<rect>`s (background, border) keep their original `width`/`height` so the visual design (4px inset border, full-size background) is preserved.

## Self-Caught Bug During the Fix

The first version of the fix used a global regex `/\swidth="[^"]*"/g` which matched **all** width attributes including the inner `<rect>` elements. This would have made the inner 4px-inset border (a `<rect>` with `width="392" height="392"`) become `width="100%" height="100%"` — overlapping the background and making the border invisible.

The fix was corrected to use `/<svg([^>]*?)\swidth="[^"]*"/` which matches only the **outer** `<svg>` tag's `width` attribute. Both rects preserve their original dimensions. Verified with a Python test against the on-disk SVG.

## Python Test of the Regex (matches the JS regex semantically)

```python
import re
svg = '<?xml version="1.0" ...><svg ... width="400" height="400"><rect width="400" height="400" .../><rect width="392" height="392" .../></svg>'
scaled = re.sub(r'<svg([^>]*?)\swidth="[^"]*"', r'<svg\1 width="100%"', svg)
scaled = re.sub(r'<svg([^>]*?)\sheight="[^"]*"', r'<svg\1 height="100%"', scaled)
# Result: outer svg width=100% height=100%, inner rects unchanged
```
