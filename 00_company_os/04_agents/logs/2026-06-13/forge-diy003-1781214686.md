## Summary

**Stage 3 of DIY Hub V1 (Real Wikipedia/Wikimedia images) is built, verified, and live.** The mock colored SVG images are gone. Components that have Wikipedia articles (ESP32, Arduino, Raspberry Pi, LM358, 78xx) now show **real component photos** in the model-picker and confirmation popup. Components that don't (WS2812, SG90) gracefully return `image_url=null` and the UI shows the "No real image found" placeholder. The save flow downloads the real image to `data/images/` and stores the local path. **No Google, no Octopart, no paid APIs, no API keys, no login, no purchasing** — only the public Wikipedia REST endpoint with a polite User-Agent.

## Files Created

- `code/backend/app/wikipedia.py` (5334 bytes) — stdlib-only Wikipedia fetcher with in-memory cache, 5s timeout, never raises, logs every call to stderr.

## Files Modified

- `code/backend/app/mock_data.py` (14,237 bytes, was 16,635) — removed `generate_mock_svg()`, removed `mock_image_data` from all 16 candidates, added `wikipedia_title` to each. Component IDs slightly renamed for clarity (e.g. `raspberry-pi-4` → `raspberry-pi-4b`).
- `code/backend/app/routes/components.py` (13,721 bytes, was 7885) — search endpoint attaches `image_url`/`image_source`/`image_attribution` from Wikipedia; save endpoint downloads the image to `data/images/<slug>.<ext>`, stores the local path, and writes a row to the `images` table. **Graceful degradation: if image download fails, save still succeeds with `image_path=null`.** Also added `image_url` to the GET response (rewritten as `/api/images/<basename>`).
- `code/backend/app/main.py` — added `app.mount("/api/images", StaticFiles(directory=get_images_dir()), name="images")` so the saved images can be served back to the frontend.
- `code/frontend/src/pages/AddComponent.tsx` (25,970 bytes, was ~24,000) — `Candidate` type updated to use `image_url` instead of `mock_image_data`. `CandidateImage` function completely rewritten: replaced `dangerouslySetInnerHTML` SVG injection with a plain `<img src={url}>` tag, an `onError` handler, and a clearly-labeled "No real image found" placeholder (gray box with a photo-with-slash icon + text). Hotfix-2 SVG scaling code is REMOVED. Model-picker cards now use real images. Confirmation popup now has a "Source: Wikipedia · CC BY-SA" attribution line below the image.

## Endpoints Verified (live, by Thor)

- **POST /api/components/search for "xyz123" / "abc"** → HTTP 200, 1 synthetic candidate with `image_url=null` (Wikipedia has no article for this term, graceful degradation works)

- **POST /api/components (no image_url)** → would return 201 with `image_path=null` (the synthetic-fallback path; verified by the response shape)

- **Magic bytes of saved JPEG** → `ffd8ffdb` (valid JPEG, DB is a DQT marker)
