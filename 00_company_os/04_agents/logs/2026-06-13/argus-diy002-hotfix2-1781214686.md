## Python Test of the Regex (matches the JS regex semantically)

```python
import re
svg = '<?xml version="1.0" encoding="UTF-8"?><svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 400 400" width="400" height="400"><rect x="0" y="0" width="400" height="400" fill="#1e3a8a" /><rect x="4" y="4" width="392" height="392" fill="none" stroke="#ffffff" stroke-width="2" opacity="0.4" /><text x="200" y="190" fill="#ffffff" font-family="system-ui, -apple-system, sans-serif" font-size="24" font-weight="600" text-anchor="middle">ESP32 DevKit V1</text></svg>'

scaled = re.sub(r'<svg([^>]*?)\swidth="[^"]*"', r'<svg\1 width="100%"', svg)
scaled = re.sub(r'<svg([^>]*?)\sheight="[^"]*"', r'<svg\1 height="100%"', scaled)

# Result:
# Outer <svg>: width="100%" height="100%"  ← scales to container ✓
# Inner <rect> background: width="400" height="400"  ← preserved ✓
# Inner <rect> border: width="392" height="392"  ← preserved ✓
```

The targeted regex correctly modifies only the outer `<svg>` tag's width/height. The background rect fills the SVG (it was already 400×400, so no change visually). The border rect stays at 392×392 with 4px inset, so the white border is still visible.

## Self-Caught Bug Note

The first version of the fix used a global regex (`/\swidth="[^"]*"/g`) that matched **all** width attributes including the inner rects. This would have collapsed the 4px-inset border rect to 100%×100%, hiding it. Forge caught this in the same report and corrected the regex to target only the outer `<svg>` tag. The final fix is correct.
