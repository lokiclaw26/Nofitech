### The 2 "CONSTANT (concerning)" items

1. **LAN IP (`#lan-pill`, `#foot-lan`).** `_detect_lan_ip()` runs once at module import (serve.py:122) and stores the result in `HOST_IP`. Within a process lifetime, the dashboard always shows the IP that was current when the server started. **Argus flag, not a Fail:** if the host's IP changes (DHCP renewal, WiFi switch, VPN connect), the dashboard would show a stale value until the server is restarted. The fix is one line (call `_detect_lan_ip()` inside `get_version()` or store nothing module-level). This is pre-existing behaviour, not introduced by Stage 15.2. **Severity: low — NOFI is on a stable home LAN, IP rarely changes mid-session, and a refresh won't help.**

2. **The 3 agents' name/role/emoji (in `AGENT_META`, serve.py:93-97).** These are the Stage 13 charter-locked identities. They are not silently constant — they are a *deliberate* design constant: the company is 3-agent by charter. **Severity: zero — this is the correct behaviour, not a leak.** Worth a `data_source: charter-locked` tag in a future schema iteration, but not a bug.

### Items that look constant but are not

- **Mode-pill (`LAN · Real Data Only`)** — constant label string, but the *meaning* it represents IS live: demo data is hidden server-side, and the demo-banner only appears when demo tasks are present. The label is a status indicator, not a data field.
- **`#foot-key` "API key: not loaded"** — string is fixed, but the boolean condition `paid.key_configured` is LIVE; the moment a key is set, the span flips to "API key: loaded" on the next refresh.
- **`api_health` always "ok"** — this is honest because the only way to read `/api/data/logs` is for the server to respond, so the API IS healthy by the time the data is returned. A dedicated health-probe (e.g. a self-ping to a child endpoint) would be over-engineering for a local-only dashboard.
- **`#last-refreshed` "auto-refresh: 30s"** — fixed string, but the actual auto-refresh is live (`setInterval(loadAll, 30000)` at html:1019). The text is a label for the live behaviour.
