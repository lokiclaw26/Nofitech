### Minor cosmetic note (non-blocking)

technical-values.md L26 states: `Total tiles | 20 × 15 = 300 tiles (40 wall + 260 floor)`

The 300 total is correct (20×15). But for a 20×15 grid with walls on the 4 edges (col 0, col 19, row 0, row 14), the corner tiles are shared, so:
- Wall tiles: 15 (col 0) + 15 (col 19) + 18 (row 0, excl. corners) + 18 (row 14, excl. corners) = **66** walls
- Floor tiles: 300 − 66 = **234** floor

The "40 wall + 260 floor" breakdown is arithmetically wrong. This is a **cosmetic arithmetic error in a non-functional spec line** (Forge will code the walls from "col 0, col 19, row 0, row 14", which IS correct, and the implementation will produce 66 walls regardless of the parenthetical). It does not contradict any other doc and does not affect the scope gate or any acceptance criterion.

**Recommendation**: Note for Forge/Thor to fix the parenthetical to "66 wall + 234 floor" in any Stage 2 cleanup pass, but do NOT block Stage 2 on this.
