## 3. HTML5 Canvas + JavaScript feasibility

**All scope items implementable with stdlib Canvas API + JS.** No external libraries, no images, no audio, no fonts. No installs.

| Spec item | Feasibility | Notes |
|---|---|---|
| 640×480 canvas, 32×32 tiles, 20×15 grid | ✅ Trivial | `<canvas width="640" height="480">` + `ctx.fillRect(x*32, y*32, 32, 32)`. |
| 60fps render via `requestAnimationFrame` | ✅ Trivial | Standard browser API. |
| Player + slime as `fillRect` squares | ✅ Trivial | Two `ctx.fillStyle` values (blue + rgb(180,40,180)). |
| HUD with `fillText` | ✅ Trivial | `ctx.font = "16px monospace"; ctx.fillText("Player HP: 5", ...)`. |
| `keydown` event handling | ✅ Trivial | `addEventListener("keydown", ...)`, supports WASD + arrows + Space + R. |
| Slime tick at 500ms | ✅ Trivial | `setInterval(updateSlime, 500)` OR accumulator inside the rAF loop using `performance.now()`. Both fine. |
| Collision: player vs wall | ✅ Trivial | Manhattan guard: `if (map[ny][nx] === 1) return;` (walls on edges per spec). |
| Collision: player vs slime | ✅ Trivial | Identity check: `if (px === sx && py === sy) takeDamage()`. |
| Game state machine (title/playing/win/lose) | ✅ Trivial | `let state = "title";` + switch in input + render handlers. |
| Restart on R (in win/lose only) | ✅ Trivial | Reset HP/positions/cooldown + `state = "playing"`. Guard: ignore R in playing/title. |
| Win at slime HP=0 / Lose at player HP=0 | ✅ Trivial | `checkWinLose()` runs each tick. |
| Greedy Manhattan AI (tie-break: N→E→S→W) | ✅ Trivial | Compute `|nx-px|+|ny-py|` for N/S/E/W, sort, pick best. |
| Attack range: 1 tile in facing direction | ✅ Trivial | `if (tx === fx && ty === fy) slimeHP--;` |
| Attack visual: yellow square for cooldown duration | ✅ Trivial | Track `attackVisualUntil = now + 400`, draw yellow square at facing tile until then. |
| Pure stdlib — no images, audio, fonts, libs | ✅ Trivial | Single self-contained `index.html` per spec. |
