### 2.3 playerAttack() logic walkthrough
```js
function playerAttack() {
  var now = Date.now();
  if (now - lastAttackTime < 400) return;        // 1) cooldown guard, exactly 400
  lastAttackTime = now;                          // 2) stamp it
  var d  = facingToDir(player.facing);           // 3) facing → unit vector
  var tc = player.col + d.dx;                    // 4) target col
  var tr = player.row + d.dy;                    // 5) target row
  if (tc < COL_MIN || tc > COL_MAX || tr < ROW_MIN || tr > ROW_MAX) {
    return;                                      // 6) wall: bail BEFORE flash & damage
  }
  attackFlash = { col: tc, row: tr, until: now + 400 };   // 7) flash on every legal swing
  if (tc === slime.col && tr === slime.row) {
    slime.hp -= 1;                               // 8) hit check: exact-match, no diagonal
  }
  render();                                      // 9) repaint
}
```
- Cooldown guard is **exactly 400** ms (line 158).
- Target is computed from `player.facing` via `facingToDir` — strictly cardinal, no diagonals.
- Bounds check uses `COL_MIN/MAX = 1/18` and `ROW_MIN/MAX = 1/13`, which match the wall ring (cols 0/19, rows 0/14) defined in Stage 1.
- The hit check is `slime.col === tc && slime.row === tr` — equality only, no distance / no diagonal.
- `slime.hp -= 1` only on a valid hit (line 170).
- `attackFlash` is set on every successful (in-bounds) swing, not just on hit.
- `render()` is the last call.
**PASS**
