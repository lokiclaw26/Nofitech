# MC-AUTO-EXECUTE-1: cron that calls delegate_task for every running_now MC-AUTO-* child

## Why
NOFI asked. "If I add a task in kanban will it AUTOMATICALLY run? I dont want to come here and tell you to do something which I have added there."

We have:
- `kanban-auto-process.sh` (cron) — moves `triage` → `ready` ✓
- `kanban-auto-dispatch.sh` (cron, 60s) — moves `ready` → spawns MC-AUTO-* child, PATCHes to `running_now` ✓
- ?? — the MC-AUTO-* child sits in `running_now` forever with no agent actually executing it ✗

This task adds the 4th piece.

## Design

The auto-execute cron will:
1. Scan `01_projects/*/tasks/` for `MC-AUTO-*` files with `kanban_status: done`
2. For each one NOT in the 120s dedup window (`.executed-<task_id>` dotfile)
3. Read the assigned agent from the frontmatter
4. Read the task title + body to construct a clear prompt
5. Call `delegate_task` (via Hermes CLI `hermes delegate <agent> "<prompt>"` or via direct Python — pick the cleanest one) to actually spawn the subagent
6. Mark the dotfile

## What the prompt should look like

The subagent gets a clean, well-scoped prompt derived from the task file:

```
You are {agent}, dispatched by auto-execute at {ts} (Dubai). Task: {task_id}.

Title: {title}

Body (verbatim from task file):
<body>

Spec from task file's "## Acceptance" section (if present):
<acceptance>

Process:
1. Read the full task file at <path>
2. Do the work
3. Write a log file to 00_company_os/04_agents/logs/YYYY-MM-DD/{agent}-<task-id>.md with what you did + a result line
4. PATCH the task to status=done via PATCH /api/data/kanban/task/<task_id> with {"status":"done","kanban_status":"done"} (use the admin token from ~/.hermes/scripts/.env.mc)
5. If you cannot complete, PATCH to status=blocked with a blocker reason in the body
6. Append a one-liner to 00_company_os/events.jsonl

You have 15 minutes. If you need more time, write a heartbeat log entry and continue. NOFI is asleep / AFK; do not ask for input.
```

## Safety rails (non-negotiable per NOFI's standing rules)

1. **Max 1 dispatch per task per 2 minutes** (dotfile mtime check)
2. **Max 3 concurrent dispatches** (don't spawn a stampede that wrecks the model API bill)
3. **Agent must be thor|forge|argus** (whitelist — refuse anything else)
4. **Task must have a `kanban_status: running_now` frontmatter** (don't run queued/done/blocked work)
5. **Skip if the agent is already executing that task** (check `pgrep -af "delegate_task.*<task_id>"` and skip if running)
6. **Log every dispatch** to `00_company_os/04_agents/logs/auto-execute.log`
7. **Kill switch**: respect a file `00_company_os/04_agents/.auto-execute-paused` — if it exists, log "PAUSED" and exit 0. NOFI can `touch` this to stop the loop without removing the cron.
8. **Per-agent rate limit**: max 6 dispatches per agent per hour (count from the log file)

## Implementation

Create `~/.hermes/scripts/kanban-auto-execute.sh` (and a copy in the repo at `01_projects/mission-control/scripts/`).

Register a Hermes cron `kanban-auto-execute` running every 2 minutes (NOT 1m — we don't need fast iteration here, 2m is fine and reduces API churn).

The script uses `hermes delegate <agent> "<prompt>"` to spawn the work. Read the existing ondemand.py + kanban-delegate.sh to see the right pattern for invoking delegate_task.
