# DIY-015 — Manual image upload for image-less candidates

**Owner:** Forge (small build) — Thor wraps up commit/push/log

## What you want (NOFI's exact words)

> "sometimes the candidates have no images from the source ... add option to add the image only manually if its missing when i select the componant from the list"

So: when a candidate is missing an image, NOFI can manually add one (file upload OR URL paste) BEFORE saving the component.

## Default Q1 (storage path): UUID-named

`data/images/<component-id>-<uuid>.<ext>` — no collision risk, audit-friendly.

If NOFI explicitly chose "B", use `<component-id>.<ext>` instead. But default is A.

## Backend changes (small, ~80 LOC)

### New endpoint: `POST /api/components/<component_id>/image`

**Two content-types supported:**

#### (a) multipart/form-data upload
- Field name: `image` (file)
- Optional field: `source_url` (string, the original URL if NOFI pasted a URL instead of uploading)
- Storage: `data/images/<component-id>-<uuid>.<ext>`
- Validates: extension in {jpg, jpeg, png, gif, webp}, size <= 5 MB
- Updates component: `image_path = "data/images/<component-id>-<uuid>.<ext>"`, `image_source = "user_uploaded"`

#### (b) application/json with URL
- Body: `{"url": "https://..."}` 
- Server downloads URL (urllib, stdlib only), validates content-type, saves to disk
- Same validation as multipart
- Updates component: `image_path`, `image_source = "user_url"`

**Response (both):** updated component JSON with `image_path`, `image_source`, `image_uploaded_at`.

**Rejects:** no component_id, no file/URL, file too big, wrong extension, URL download fails, URL content-type not image.

### Helper function in `app/`:

- `save_uploaded_image(component_id, file_content, source_ext)` → (path, error)
- `download_and_save_image(component_id, url)` → (path, error)
- `validate_image(content, ext)` → (ok, error_msg)

## Frontend changes (small, ~150 LOC)

Props:
- `componentId: number`
- `onImageAdded: (imagePath: string) => void`

Behavior:

- After save: hide the picker, show the new image + "Change image" / "Remove" buttons
- Error states: too big, wrong format, download failed
