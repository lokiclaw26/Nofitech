# DIY-008 — Stage 8: Operator UX bug fixes

## Why
NOFI (operator) tested Stage 7 and found 2 critical UX issues:

### Bug 1: "Enter manually" missing from confirm dialog
The "Enter manually" button is ONLY visible in the empty-state dialog
(when live search returns 0 candidates). When live search returns 1+
candidates, the flow goes directly to "Confirm component details" and
shows ONLY Cancel + ADD TO DATABASE. If NONE of the candidates match
what the operator actually wants to add, they're forced to cancel the
entire search and start over.

**NOFI's exact words:** "the add manually doesnt work .. no button for
manual entry is there .. i think this button should be FOR EACH AND
EVERY SEARCH even if you find the results .. maybe the one i have is
not in your list"

**Reproduction:** type "testestestes" → live search returns 1 bogus
PlatformIO candidate → confirm dialog shows → no way to "just type it
myself" → operator is stuck.

**Fix:** Add an "Enter manually instead" button to the confirm dialog
AND the model-picker dialog. The button is always visible regardless of
how many candidates came back. Clicking it:
- Closes the current dialog
- Resets the pickedCandidate
- Opens the manual entry dialog
- Pre-fills nothing (operator starts fresh)

### Bug 2: No view toggle on Inventory page
Inventory page is cards-only. NOFI wants:
- "List view" = small image + name + model + qty + location (compact, scannable)
- "Card view" = large image + all details (current behavior)
- Toggle in the top-right of the inventory page
- Preference persisted in localStorage so it survives reloads

**NOFI's exact words:** "add the view option .. to be list or cards ..
so maybe i wanna look at my componants in a list and info with small
image.. or cards with larg images"

### Out of scope (NOT fixing in this stage)
- The "many old records have no image" issue. NOFI noted this but it's
  a data issue, not a code issue — those records were saved before
  Stage 5 when there was no image lookup. The Operator can either:
  1. Delete them manually (not in scope yet)
  2. Wait for edit/delete feature (Stage 9+)
  3. Re-add them with the new live search (which now has real images)
  The Inventory page ALREADY shows the "No image" placeholder correctly.

### Fix 1: "Enter manually" button in confirm dialog
- Add Button "Enter manually instead" to the confirm dialog footer
- Position: between Cancel and ADD TO DATABASE, with variant="ghost"
- data-testid="btn-enter-manually-from-confirm"

  the user sees multiple candidates and none match)

### Fix 2: View toggle on Inventory
- Add toggle UI to the top-right of the inventory page (next to "+ Add Component")
- Two view modes: "cards" (current) and "list" (new)
- Use a small button group with icons (grid icon + list icon)
- Persist preference in localStorage key: "diy-hub-v1-inventory-view"
- Default to "cards" if no preference is set
- List view layout:
  - Each row: small 48x48 image area | name (bold) | model (small) | mfr | qty + location | category chip | source badge | confidence badge
  - Denser, more rows per page
  - No card border, just a row separator
