# DIY-007 — Stage 7: Inventory List + Manual Entry

## Why
Stages 1-6 gave us: live search, real images, vendor scraping, save flow.
What we DON'T have yet:
1. A way to SEE the saved components. The Inventory page is currently a
   placeholder empty state.
2. A way to add a component when the live search returns 0 candidates.
   The "Enter manually" button in the empty-state dialog (Stage 5)
   currently just closes the dialog — there is no form.

## NOFI approval (verbatim)
> "NOFI approves Candidate A + B for Stage 7.
> Build Inventory List page, add Manual Entry fallback form.
> Do not build edit/delete yet. Do not tweak live search yet. Do not
> add Idea Lab yet."

## Scope (locked — DO NOT EXPAND)

### Scope A — Inventory List page
- Read all saved components from SQLite via GET /api/components
- Render as a responsive grid of clean cards
- Each card shows:
  - real image if available (else gray "No image" placeholder)
  - name
  - model_number
  - manufacturer (if available)
  - quantity
  - location (if set)
  - category
  - voltage (if set)
  - interfaces (as chips, if any)
  - tags (as chips, if any)
  - confidence badge (color-coded)
  - source attribution ("live", "adafruit", "mock_fallback", "manual")
- Search bar (single text input, filters by name, model_number, category,
  location, tags, notes — case-insensitive substring match, client-side)
- Category filter (dropdown of all distinct categories present in DB)
- Empty state: "No components yet — add your first component."

### Scope B — Manual Entry fallback form
- "Enter Manually" button accessible from the Add Component empty-state
  dialog (where the "Try offline mock fallback" button already is)
- Manual form fields (per NOFI brief):
  - name (required)
  - model_number (required)
  - manufacturer (optional)
  - category (dropdown, required — defaults to "Other")
  - quantity (number, required, default 1, >= 1)
  - location (text, optional)
  - voltage (text, optional)
  - interfaces (chip-input, comma-separated, optional)
  - key_specs (chip-input, comma-separated, optional)
  - tags (chip-input, comma-separated, optional)
  - description (textarea, optional)
  - image_url (text, optional — if provided, backend will download it)
- Save through the same POST /api/components endpoint with source: "manual"
- Backend already accepts source: Optional[str] = "live" (Stage 5).
  We just need to make sure "manual" is also accepted.
- If image_url is provided, existing download_image() in routes/components.py
  downloads it to data/images/<slug>.<ext> (Stage 5 implementation)
- If no image_url, save with image_path = null (already supported)
- Show success animation, then close + reset form

## Frontend changes (the bulk of the work)

### Inventory.tsx (full rewrite)
- useState for: components, searchQuery, selectedCategory
- useEffect to fetch /api/components on mount
- Filtered list = components.filter(matchesSearch && matchesCategory)
- Each card: 200x200 image area, name (bold), model_number (small),
  manufacturer, qty + location, category, voltage, chips for interfaces
  and tags, source badge, confidence badge
- Search bar: <input data-testid="inventory-search" />
- Category filter: <select data-testid="inventory-category-filter" />
  with options: "All" + every distinct category from the loaded data
- Empty state: when filtered list is empty (after search/category filter
  or when DB has 0 rows), show different message
- Loading state: show "Loading..." while fetch is in flight
- Error state: show "Failed to load inventory" with a Retry button

### AddComponent.tsx (minor)
- Add a "manual" state: when set, render the manual form modal instead
  of the empty-state dialog
- Empty-state dialog now has 3 buttons: Try offline mock fallback,
  Enter manually, Cancel
- Clicking "Enter manually" sets manualOpen = true
- Manual form modal: same dialog shell, contains the form fields
- Form submit calls POST /api/components with source: "manual" and the
  form values, using the same handleSave logic
