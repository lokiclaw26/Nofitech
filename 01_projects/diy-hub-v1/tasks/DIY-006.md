# DIY-006 — Stage 6: Vendor Site Scraping (Adafruit + Pololu)

## Why
Stage 5 (DIY-005) replaced the mock-only catalog with 5 free public sources
(Wikimedia Commons, Wikidata, Wikipedia, PlatformIO, GitHub). These cover
most well-known components but return 0 candidates for niche DIY parts like
BME680, INA219, MAX6675, TP4056, BME280, DS18B20, etc.

NOFI (operator) has now approved adding 2 vendor sites (Adafruit, Pololu)
as additional live sources. These are the most popular DIY hobbyist vendors
and have real product pages with real images, real descriptions, and real
datasheet links — and their robots.txt EXPLICITLY ALLOWS scraping of
product pages.

DDG HTML scraping was proposed and REJECTED (DDG's robots.txt disallows
/html). SparkFun was proposed and REJECTED (their search results are
JS-rendered, no product URLs in static HTML). Vendor sites that work via
static HTML are the right answer: real product data, allowed by the site
owners, no key, no login, no payment.

## NOFI approval (verbatim)
> "NOFI approves vendor site scraping.
> Approved sources: Adafruit, SparkFun, Pololu.
> Rules: Respect robots.txt and crawl-delay. Use cache to avoid repeated
> requests. Use vendor pages only for component lookup, real images,
> descriptions, datasheet links, and source links. Do not scrape
> aggressively. No DDG scraping. No Google Custom Search. No Octopart.
> No paid APIs. No API keys. No login. No purchasing actions. Do not
> display price/stock for now. Do not auto-save anything without ADD TO
> DATABASE confirmation."

## Pre-implementation findings (verified live from this machine)
- **Adafruit**: ✅ works. Search `/search?q=<query>` returns product links. Product pages have og:title, og:description, og:image, AND a real datasheet PDF link in the page body.
- **Pololu**: ✅ works. Search `/search?query=<query>` returns product links. Product pages have og:title, og:description, og:image. Search returns 0 results for components they don't carry (e.g. BME680) — this is correct.
- **SparkFun**: ❌ REJECTED. Search results are JS-rendered; the static HTML has no product URLs. Cannot be reliably scraped without a headless browser, which would violate the "no aggressive scraping" rule.

SparkFun is parked. If a component is in SparkFun's catalog but not in
Adafruit/Pololu/Wikimedia, the operator can use the "Enter manually" form
(parked for Stage 7+).

## Goal
Add 2 live fetchers (Adafruit + Pololu) to ``app.live_search`` so that
niche components not covered by Wikimedia/PlatformIO/etc. can still
return real product candidates with real images, real descriptions, and
real datasheet links.

## Hard rules (locked)
- Respect robots.txt for each site
- Respect crawl-delay: 1s minimum + 1-hour cache per query+vendor
- Use cache to avoid repeated requests (vendor cache TTL = 1 hour)
- Vendor pages ONLY for: component lookup, real images, descriptions,
  datasheet links, source links
- NO DDG scraping
- NO Google Custom Search
- NO Octopart
- NO paid APIs
- NO API keys
- NO login
- NO purchasing actions
- NO price/stock display
- NO auto-save — operator must click ADD TO DATABASE

## Per-vendor details

### Adafruit
- robots.txt: ``Allow: /`` for *, only disallows /api/, /ajax/, /cache/, /download/, /i/, /includes/, /tmp/
- Search URL: ``https://www.adafruit.com/search?q=<query>`` returns HTML list
- Product URL: ``https://www.adafruit.com/product/<id>``
- og:title, og:description, og:image always present in product pages
- Datasheet link: any ``<a>`` with text containing "datasheet" in the page body
- Crawl-delay: not specified → 1s minimum + cache
- Identify via User-Agent: ``DIY-Hub/1.0 (https://github.com/nofitech; contact: ops@nofitech.local)``

### Pololu
- robots.txt: disallows /search, /search/, /account, /u, /feedback, etc. — but ALLOWS /product/ and /catalog/
- Search URL: ``https://www.pololu.com/search?query=<query>`` (returns "no results" page if not found)
- Product URL: ``https://www.pololu.com/product/<id>``
- og:image, og:description present in product pages
- Crawl-delay: not specified → 1s minimum + cache
- Identify via User-Agent

## Polite-crawler policy

- 1-second minimum pause between requests to the SAME vendor
- 1-hour cache per query+vendor (vendor cache, distinct from the 60s
  primary-source cache)
- User-Agent identifies the project
- NEVER scrape more than 1 product page per query (1 search result +
  1 product page = 2 HTTP requests max per vendor per query)
- NEVER follow pagination, never scrape /search for the next page
- If a vendor returns 0 search results, give up immediately (no retry)
