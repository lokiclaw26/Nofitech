6IoT Setup — Cameras, Microcontrollers, Sensors

Critical for your fleet. Most 2.4 GHz IoT devices are picky. Do this during the install, not after.

⚠️ Why IoT setup is different from phone/laptop Wi-Fi

Phones, laptops, TVs support both bands and roam fine. IoT devices (ESP32, ESP8266, cheap IP cameras, Tuya, Sonoff, some sensors) are often single-band, low-power, sticky-client. They hate band-steering and hate weak signals. Treat them as a separate network concern.

Recommended TP-Link Deco IoT configuration

  1. Keep a single SSID (Ahmad_Not) for all your devices. Don't split into 2.4/5 GHz SSIDs — the Deco's band-steering works well for non-IoT devices, and TP-Link's IoT-aware mode (Deco app → More → IoT) auto-detects single-band devices and pins them to 2.4 GHz without manual config.
  2. Enable the "IoT Network" in Deco app (More → Wi-Fi Settings → IoT Network). This creates a dedicated 2.4 GHz-only SSID like Ahmad_Not_IoT with device isolation (cameras can't talk to your laptop, just to the internet). Use this only for devices that absolutely refuse to connect to the main SSID.
  3. Channel width = 20 MHz on 2.4 GHz. Don't use 40 MHz — it crowds the band, breaks older IoT clients. Set in Deco app → Wi-Fi Settings → 2.4 GHz → Channel Width.
  4. Channel = 1 or 11 (not 6). Channel 6 is the default for most APs in your area — picking 1 or 11 gives you a clean neighborhood. Set in Deco app → Wi-Fi Settings → 2.4 GHz → Channel.
  5. WPA2/WPA3 mixed mode (not pure WPA3). Many IoT devices only support WPA2. Pure WPA3 will lock them out.
  6. Disable "Fast Roaming" / 802.11r in the Deco app for IoT-heavy setups. (More → Advanced → Fast Roaming). Sticky clients (cameras especially) prefer stable over fast.
  7. Place nodes high and central — 2.4 GHz propagates better than 5 GHz but is still hurt by walls/floors. The wall mounts (AED 110 add-on) really help here.

Per-device-type checklist

DeviceBandSSIDNotes

If a device won't connect — debug ladder

  1. Forget the network on the device, re-add with password
  2. Try the Ahmad_Not_IoT SSID (2.4 GHz only)
  3. Move the device closer to the nearest Deco node (within 5 m)
  4. Check WPA mode is mixed WPA2/WPA3, not pure WPA3
  5. Check channel width is 20 MHz, not auto/40 MHz
  6. Reboot the device + the closest Deco node
  7. Factory-reset the device, retry from scratch
ℹ️ Why not just disable band-steering and split into two SSIDs?

You can, and for very stubborn IoT devices it's the nuclear option. But: (1) you lose seamless roaming for phones, (2) TP-Link's "IoT mode" already handles this well in 2026 firmware, (3) split SSIDs make it harder to find the right device on your network. Try the unified SSID first, fall back to Ahmad_Not_IoT only for devices that genuinely refuse.

7Install Plan — Step by Step

Total time: ~45 minutes from "package opened" to "everything working". You need a phone with the TP-Link Deco app installed.

Step 1 · Unbox

Open the 3-pack box

All 3 nodes are identical. Each has: 1× Deco unit, 1× power adapter, 1× flat Ethernet cable. Pick one node to be the main — that one goes near your Etisalat/du ONT (the small box on the wall where the fiber comes in).

Step 2 · Connect main node

Plug main node into ONT

Use the included Ethernet cable: one end → ONT's LAN port, other end → main Deco's WAN port (any port will do — Deco auto-detects). Power on. Wait ~2 minutes for boot (light goes from yellow → green/blue).

Step 3 · App setup

TP-Link Deco app does the rest

Open the app, sign in / create TP-Link account, scan QR code on main node. App asks for ISP type — pick Etisalat or du (whichever you have). WAN = DHCP. Done.

Step 4 · Add the other 2 nodes

Placement matters

Plug node 2 in upstairs or in the far room from the main node. App walks you through placement: it'll tell you if signal is weak, OK, or great. Repeat for node 3. Target: signal strength "great" from each node to the main.

💡 Tip: Don't put nodes directly above each other on different floors — diagonal placement reduces floor/ceiling attenuation.

Step 5 · Set your SSID

Keep "Ahmad_Not"

Set Wi-Fi name to Ahmad_Not — same as your current SSID. Set a strong password. WPA3 if all your devices support it, WPA2/WPA3 mixed otherwise.

Step 6 · Decommission old gear

Unplug Archer C7 and Asus repeater

Once the new mesh is up and your phone is on the new SSID with internet working, unplug the Archer C7 (power + Ethernet from ONT). Then unplug the Asus repeater. Don't factory-reset yet — leave them as-is for 1 week as fallback in case something goes wrong.

Step 7 · Wait for migration

30 minutes, that's it

Because you kept the SSID name, all your devices will automatically reconnect to the new mesh. Some may need a toggle (turn Wi-Fi off/on once). Phones, laptops, smart home devices, TVs — they all figure it out within ~30 min.

8Post-Install Verification

Run through this checklist after the install. If any item fails, check the troubleshooting section below.

Quick troubleshooting

  • "Phone won't connect to Ahmad_Not" → Forget the network on phone, re-add manually with password
  • "5 GHz shows 2.4 GHz only" → Device is too old for 5 GHz, or stuck on 2.4 — toggle Wi-Fi off/on
  • "Speed is slow in far room" → Node is too far. Move it closer to main or add a 4th Deco node (X55 supports it)
  • "Deco app says weak signal between nodes" → Move nodes closer, or connect them via Ethernet if you have in-wall wiring

9Pro Tips From the Network Diagnostic Guide

🏠 Placement

  • Place nodes on inner walls, not exterior masonry (UAE villas have lots of concrete — kills signal)
  • Don't hide nodes in cupboards or behind TVs — WiFi hates enclosed spaces
  • Aim for nodes at chest height or higher (wall mounts help)

🔌 Wired backhaul = best performance

If you have Ethernet cabling in your walls (Cat5e or better), connect nodes 2 and 3 via Ethernet instead of relying on wireless mesh. This turns the mesh into a "wired backbone + wireless access" setup — fastest possible.

📱 Use the Deco app, not the web UI

Deco is managed 100% from the app. The web UI is minimal. Install the app, enable notifications, and let it auto-update firmware.

🔒 Security

  • Enable WPA3 in Deco app if all your devices support it (most 2019+ do)
  • Set a guest SSID for visitors (different password, isolated from main)
  • Turn on HomeShield if available — built-in malware/phishing filter
  • Change the admin password from default (the app walks you through it)
⚠️ Do NOT enable "Bridge Mode" unless you know why

If you put the Deco in bridge mode you lose half its features (AI roaming, security, parental controls). Leave it in Router Mode — that's the default.
