4Why WiFi Feels Slow — Symptom Breakdown

Three independent things are stacking up. Each one alone wouldn't ruin your network. All three together = "feels like 50% speed drop".

Symptom 1: Your devices see 2 different networks

BandSSID
2.4 GHzNoRed5
5 GHzAhmad_Not

Your phone connects to NoRed5 (2.4). You walk to the room where 5 GHz is strong. Phone stays on 2.4 GHz because it doesn't know Ahmad_Not is the same router. → slow.

Symptom 2: Channel 6 is the worst in Dubai apartments

Channel 6 is one of 3 non-overlapping channels (1, 6, 11), which means every router in your building is also on it. The Archer C7's default 2.4 GHz channel is 6 = stomp fight with 30+ neighbors. Plus the gateway at .1 is probably also on 6 = internal stomp fight too.

Symptom 3: Asus "repeater" is half-bandwidth by design

When the Asus is in "repeater" mode and connects wirelessly to the Archer C7, it has to:

  • Receive packets from the Archer on 1 radio
  • Forward them out on the same or different radio

Result: ~50% of the bandwidth is wasted on the backhaul + ~10–30 ms extra latency per hop. You see this in the 50–200 ms ping to the gateway for Asus-connected clients.

5Correction to My First Diagnosis

⚠ I had it wrong. Here's the correction.

My initial theory was "double-NAT is killing your network" because the Archer C7 was on the same 192.168.0.x subnet as the gateway. The screenshot you sent proved that wrong:

FieldValueWhat it means
WAN IP0.0.0.0🚨 WAN port is unplugged
WAN Status"WAN port is unplugged!" (red)Confirmed
Traffic Statistics0 bytes / 0 packetsNo traffic has flowed through this router
LAN IP192.168.0.2On same subnet as gateway
Operation Mode"Wireless Router"Misleading label — WAN unplugged means it's effectively AP

Since the WAN is unplugged and zero traffic flows through it, the Archer C7 is functioning as an access point even though the menu says "Wireless Router". The ethernet cable from the gateway must be plugged into one of the Archer's LAN ports, not the WAN port.

💡 Don't fix what isn't broken
Do NOT change the Archer C7's Operation Mode from "Wireless Router" to "Access Point" — that would re-enable NAT and break things. The current state is fine.

6The 4 Real Problems

Problem A — Different SSIDs on 2.4 vs 5 GHz (the #1 roaming/speed killer)

Your devices can connect to either, but not both at the same time, and they don't know they're the same router. When your phone is connected to NoRed5 and you walk to the 5 GHz-only zone, it stays on 2.4 GHz = slow.

Problem B — 2.4 GHz on Channel 6, in Dubai that means CONFLICT

Channel 6 is the most popular in dense apartments. You're competing with everyone in your building. The gateway is probably also on 6 → internal stomp fight.

Problem C — 5 GHz auto-channel is set to 157, with WDS disabled

Auto channel on 5 GHz causes "WiFi drops randomly" — a neighbor's radar pulse or new AP can trigger a channel switch = 5–10 sec disconnect. Lock it to a fixed channel.

Problem D — The Asus "repeater" is in repeater mode (wireless backhaul)

Same MAC roaming between .12, .13, .21 = classic repeater behavior when parent AP signal wobbles. Half bandwidth + extra latency for any device connected to it.

7Step-by-Step Fix Plan (Ranked by Impact)

Do these in order. Each step gives an immediate win. The full plan takes 30–45 minutes.

Step 1

Match the SSIDs (fixes roaming + confusion)

Goal: One network Ahmad_Not across both bands. Devices auto-pick the best band.

  1. Open http://192.168.0.2 (you have this open already from the screenshot)
  2. Click Wireless 2.4GHz in the left menu
  3. Change Wireless Network Name (SSID) from NoRed5 to Ahmad_Not
  4. Click Save
  5. Click Wireless 5GHz
  6. Confirm SSID is Ahmad_Not
  7. Click Save

Result: All devices see ONE network. They auto-use 2.4 for distance, 5 for speed.

Step 2

Move to non-overlapping channels (fixes interference)

Goal: 2.4 GHz off channel 6. 5 GHz on a clean fixed channel.

  1. Archer 2.4 GHz → set channel to 1 or 11 (not 6)
  2. Archer 5 GHz → set channel to 36 or 52 (clean UNII-1 band)
  3. Then log into the main TP-Link gateway at http://192.168.0.1
  4. Set its 2.4 GHz to a different one of 1/6/11 than the Archer

Target state:

Device2.4 GHz channel5 GHz channel
TP-Link gateway (.1)1152
Archer C7 (.2)136

(Or vice versa — just make sure they're different.)

Step 3

Lock the 5 GHz to a fixed channel (not Auto)

Auto channel on 5 GHz is the most common cause of "WiFi drops randomly." Set it to a fixed channel and it stays put.

If you did Step 2, this is already done (you set the channel, not Auto). If not, do it now.

Step 4

Deal with the Asus repeater (3 options, in order of preference)

Option A — Run a $5 ethernet cable (best)

  1. Run an ethernet cable from the switch (or the Archer C7's LAN port) to the Asus
  2. On the Asus, set Operation Mode → Access Point
  3. Full bandwidth at the far end. No wireless backhaul penalty.

Option B — Media Bridge mode (if you can't cable)

  1. On the Asus, set Operation Mode → Media Bridge (a.k.a. Client Bridge)
  2. Asus still uses Archer wirelessly as backhaul
  3. But instead of re-broadcasting as a new AP, it just provides 4 ethernet ports at the far end
  4. Plug far-end devices into it with cables
  5. No half-bandwidth penalty. No WiFi re-broadcast.
⚠ Trade-off
You lose the ability to put WiFi clients on the Asus this way. But the connection to the rest of the house is solid.

Option C — Keep repeater mode but force 5 GHz backhaul (worst option)

  1. In Asus admin, Repeater mode → set the 5 GHz uplink to a different channel than Archer's 5 GHz
  2. Asus uses 5 GHz to talk to Archer, 2.4 GHz for clients
  3. Half the 5 GHz bandwidth is wasted on backhaul
Step 5 (optional)

Reboot everything in order

After making all the changes, a clean reboot clears stuck leases and channel conflicts:

  1. Unplug all 3 boxes (gateway, Archer, Asus) from power
  2. Plug in the main gateway → wait until fully booted (1–2 min, lights stable)
  3. Plug in the Archer C7 → wait 1 min
  4. Plug in the Asus (if you kept it) → wait 1 min
  5. Reconnect your phone/laptop to Ahmad_Not and test

12Security Notes

🔒 Why I don't take your router password

Even from you, the owner. My reasons:

  • This conversation is logged in a system other agents can see (and one day, the public)
  • A leaked router password = attacker owns your network forever
  • You changing the password after I touch it is one extra step for you; a network breach is hundreds of steps
  • Settings changes should always be you, on the keyboard in front of the router

What I CAN do

  • Read live network state (latency, device list, signal strength)
  • Diagnose and recommend
  • Verify after you make changes
  • Read screenshots of admin UIs and tell you what to click

What I will NEVER do

  • Log into your router admin
  • Accept the password (even via screenshot, even if you insist)
  • Run any "helper" tool that needs it
  • Modify any device config remotely

Bonus security tip (worth doing while you're in the admin UI)

  • Check Admin → Remote Management is DISABLED (off). Default is usually off, but verify.
  • Check Wireless → Password is WPA2-PSK or WPA2/WPA3 mixed. Not WEP or "no password".
  • Change the router admin password from default (e.g., admin/admin). Even if no one can reach the admin from the internet, defense in depth.
  • If your firmware has an update available, install it. Especially for the Asus repeater.
